Slide 5: The Bronze Layer (Deep Dive)

Design Tip: Use this consistent 4-row table layout for Slides 5, 6, and 7. Paste this directly into your 2-column PPT table.
Core Objective To act as a resilient, immutable historical archive of source data, exactly as it was received from operational systems.
Processing Steps
  • Ingest raw CSV batches (Loans, Demographics, Financials, Clickstream).
  • Inject ingestion_timestamp metadata for auditability.
  • Convert flat files to highly compressed columnar Parquet format.
Data Quality & Logging Data is intentionally left uncleaned to prevent premature data loss. Process outcomes and row counts are recorded in the central logs/pipeline.log.
Output & Volume 4 distinct Bronze Parquet tables.
Key Volumes: Loans (137.5k), Clickstream (215.3k), Attributes & Financials (12.5k each).

Slide 6: The Silver Layer (Deep Dive)

Design Tip: Same consistent table structure, focusing heavily on the cleaning and consolidation logic we engineered.
Core Objective To clean, filter, and conform the raw data into a reliable, enterprise-grade schema ready for complex joins.
Processing Steps
  • Imputation: Replaced anomalies in Age (7.9%) and Num_of_Loan (4.5%) with dataset medians.
  • Outlier Handling: Winsorized Annual_Income at the 99th percentile ($195,000) to cap extreme skew.
  • Consolidation: Executed an INNER JOIN on Attributes and Financials on (Customer_ID, snapshot_date) to create a unified Customer Profile.
Data Quality & Logging Automated thresholds trigger WARNING logs if feature anomalies exceed 5% (e.g., Age column flagged).
Output & Volume 3 distinct Silver Parquet tables.
silver_customer_profile successfully consolidated to exactly 12,500 highly-cleansed rows.

Slide 7: The Gold Layer (Deep Dive)

Design Tip: Emphasize the temporal logic and dataset splitting here. This is the crown jewel of the pipeline.
Core Objective To generate a unified, point-in-time accurate Feature Store and Label Store directly consumable by Machine Learning models.
Processing Steps
  • Timeline Spine: Anchored all joins to the 137,500 daily loan snapshots.
  • ASOF Joins: Forward-filled Customer Profile data to precisely match the loan snapshot date (Zero Leakage).
  • Windowing: Calculated 90-day rolling sums and averages for all 20 Clickstream features.
Temporal Splitting Engineered a dataset_split indicator column inside the Feature Store. Snapshots before May 2025 are flagged as Train (123k rows), while future events are held out as OOT (14k rows).
Output & Volume gold_feature_store containing 137,500 fully-engineered rows, paired with the gold_label_store targets.

Slide 9: Pipeline Execution & Reproducibility

Design Tip: Expand your 4 infrastructure points into sharp, business-value bullet points. You can lay these out in a 2x2 grid on the slide.

🐳 Docker Containerization

Encapsulates all dependencies, OS libraries, and Python environments. Guarantees 100% execution consistency across local dev, testing, and production servers, eliminating "works on my machine" bottlenecks.

⚡ Apache Spark (PySpark)

Utilizes a distributed computing framework for data transformations. Ensures the pipeline architecture can seamlessly scale out as data volumes grow from megabytes to terabytes without code rewrites.

🐙 Git Version Control

Provides a strictly audited, time-stamped history of all data engineering logic changes. Enables peer review and immediate rollback capabilities if a pipeline update introduces data corruption.

🗄️ Parquet Storage Format

Highly optimized columnar storage format. Significantly reduces disk footprint through advanced compression and drastically speeds up I/O reads during downstream Machine Learning training.
